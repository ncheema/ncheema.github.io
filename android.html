Android Security

Posted on March 10, 2015

As of early 2015, Android operating system has over 50% market share of Smart Phones Operating System; which makes it vulnerable to malicious attacks due to its popularity.For the last two years I got the opportunity to work in the field of android application development. One of the recent areas I worked was resolving issues related to Android Security while my summer 2014 internship. I came across certain flaws in Android Security, which are neglected by the Android users daily, which can cause great damage. Data and privacy being highly valuable we are going to focus on how data and privacy can be vulnerable from a user point of view and how to prevent it. Reason being that most of the time malicious attacks happen due to lack of knowledge from a user.

Google play store recent malicious software tool does have malicious software checks running on your device to check for attacks while the application is running. Moreover each application goes through multiple malware checks before getting published on the Google play store. Along with that Android by default doesn’t allow any application to read or write to private data or data of other applications. Where does the security flaw occur? Android allows applications to ask for certain permission to access core features such as camera, data, microphone, and user of the application has to decide whether it wants to grant those permission. While downloading an application, user don’t pay close attention to the permissions they are grating to the application. For example an application might ask permissions to access your microphone, now that application can access microphone anytime it wants, and send the recordings to whomever it wants. Now same scenario applies while giving permission to access photos and data. Here is an example of permission page. (These are some of the permission asked by facebook application)

Moreover user use third party unauthorized application stores to download the paid apps for free, where no malware detection check takes place and user has a high chance of downloading malicious software. To Disable the option for dowloading application from unknown sources do the following on your Android device: disable Unknown sources under Settings > Security on the device.

First reaction of most users is to go download anti-virus software on their mobile device to protect themselves. However bad news is anti-virus wouldn’t protect users from above scenarios. Reason being that anti-virus software usually search for patterns such as when an application trying to get root/admin access without the permission of the user. However in the above scenario user already granted permission to the application at the beginning of the download. So what should users do to protect themselves?

I have encoutered these issues from a developer side, where I had to protect the user from the malicious attacks menitioned above. Following is a video showing one of the solution I used to protect user data.



Now how can an average users protect themselves? First part of the solution is instead of downloading applicaitons from third party sources; download the application from Google play store due to its malware detection software on the play store. Along with that a user should double check permissions for the application before granting permission. If an application is designed to show current temperature of the city, it shouldn’t have access to your camera, and microphone. Moreover user should research how certain permissions are being used by the application, and turn off the unnecessary permissions. Besides that users should only allow certain permission to the application while the application is being used/active, rather than permission being active all the time. Android open source model does create certain security risks but can be tackled if taken right security precautions as mentioned above by its users.
