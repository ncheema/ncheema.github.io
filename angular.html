Web Application technologies - Angular.JS [Part I of II]

Posted on March 15, 2015

This is the part I for the two part series where we are going to decide back-end and front-end technologies for the following web applciation. Our application is a Medical office patient check-in service. We are going to send notifications to the doctor once the patient check-in. Along with that we are going to give patient the ability to fill up forms upon check-in.

First we are going to discuss technology to be used on the front-end side. Angular.JS is one of the hottest front-end framkeworks at the moment. Angular would be a good fit for our applciaiton. Just like any other framework AngularJS has its pros and cons, but given its functionality it’s a good fit for Application described above. Our application requires two main functionalities on the front-end: Login and Register. Both features are interaction using forms, and can be accomplished using dynamic single page web application. Form elements re-implements the same structure and functionality repetitively. This problem can be simply solved by using AngularJS directives, which will result in fast implementation and reduce redundant code. For example AngularJS gives the ability to write a custom tags, which compiles to the original complicated code and can have its own scope and internal logic.

Developing the single page web application can be done very easily using ng-route (mange routing of web application) and ng-view directives in AngularJS. AngularJS gives us the ability to only change elements, which requires a change. Now why is that good in Peter’s application? When a user would check-in using application we can keep the same header, menu, background images and only change the elements which requires the change; which in our case would be resetting the form and showing the user a pop-up message, to indicate successful check-in. Along with that since showing a popup would be a distinct functionality it be great to have distinct controller for that functionality. AngularJS provides this functionality to organize application logic into modules and make them work together using dependency injection.

AngularJS has the ability to divide app into smaller states using separation of templates, controller and directives, which increases workflow, less code to test and maintain; which increases overall productivity. Since we need to deploy this application in within next eight weeks, AngularJS would be a great fit.
