Android Work

Posted on March 16, 2015

The following is my critique of the article “How will Android’s new “Work” features affect enterprise mobility Management?“ by Jack Madden. In the article author talks about the Android’s new enterprise mobility management frameworks called Android Work and discusses the core features and capabilities of the new feature.

I do agree with the author that all the hype about Android Work having the Samsung Knox API is false. Since Samsung later last year said that, "not everything from Knox will be contributed to Android". There could be a case where Samsung is planning on contributing the knox EMM (Enterprise Mobility Management), which is a third party EMM. Author claims that user should be able to separate the work profile from personal profile, and work profile would have targeted application on Google Play store which would be managed by IT. However this doesn’t solve the major issues of downloading applications from third party sources. Even though restricting Google Play Store would provide great management, however it would still lacks the ability to secure the device from malicious software over the Internet. Another thing to note is Android Play Store will let enterprise accounts license/download application in bulk, which could save companies large sums of money in the long run.

Now one of the key features which do stand out in Android Work is the separation of data. IT administers are going to have the ability to create a secure work profile on the device which would separate the data and applications in work and personal profile. But we have yet to see how secure are the boundries between the work and personal profile. Another issue which author does mention but doesn’t focus on is what happens to the android devices which wouldn’t be able to support Android work? Not all the current Android Devices are compatible with Android Work, which means companies/individuals would need to invest in new hardware.

Along with that another issue which author’s doesn’t focus on is that Android is giving more power to individual EMM vendors, which means that each MDM vendor is building their own proprietary MDM agent app, Now this could lead to certain security issues. But we would have to wait and see how that decision would turn out. In the closing of the article author does mention alternative solutions using via software. This could be a decent solution, but what the corporate world need is a solid security solution from a lower level of OS; which could be provided by Android Work, but it's too early to run to conclusions. Android Work is still in its early stage and lots of changing are going to happen in the coming months.
